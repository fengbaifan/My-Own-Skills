# S05 检索策略解析

## 定位

清洗流水线**噪音预判技能**。检索策略决定了数据集的噪音结构——本技能通过分析检索式，**在清洗之前**就建立噪音类型的预期模型，使后续清洗有的放矢。

## 前置依赖

- S03-研究背景问询 已完成（提供检索式和研究主题）
- S04-映射表构建 已完成（提供概念群命中分布）

## 分析框架

### 第一步：检索式结构拆解

将完整检索式拆解为：

```
[检索式] → [概念群] → [检索词] → [实际命中词形]
```

对每个概念群标注：
- 检索方式：精确短语 / 通配符 / 布尔组合
- 检索字段：TS (全文) / TI (标题) / AB (摘要) / DE (关键词) / 全字段
- 命中篇数和占比

### 第二步：通配符风险分析

对每个使用通配符的检索词，分析可能命中的词形：

| 分析维度 | 问题 |
|----------|------|
| **词根扩展** | 通配符实际命中了哪些词形？ |
| **跨语言同形** | 目标词在其他语言中有什么含义？ |
| **跨学科同形** | 目标词在其他学科中有什么含义？ |
| **缩写冲突** | 目标词是否是某个缩写的一部分？ |
| **命名冲突** | 目标词是否是品牌名/地名/人名？ |

**风险等级评估：**

| 等级 | 定义 | 预期噪音比例 |
|------|------|-------------|
| 🔴 高风险 | 通配符命中大量无关词形 | >30% |
| 🟡 中风险 | 存在已知同形异义 | 10-30% |
| 🟢 低风险 | 精确短语，特异性高 | <10% |

### 第三步：噪音类型预测

基于检索策略分析，预测可能出现的噪音类型：

**三层噪音预测模型：**

| 层级 | 类型 | 核心问题 | 典型来源 |
|------|------|----------|----------|
| **第一层** | 词形噪音（通配符溢出） | 命中词根本不是目标词 | 通配符扩展到无关词形 |
| **第二层** | 主题噪音（同词异义） | 词对了但含义不同 | 跨学科/跨语言同形异义 |
| **第三层** | 边界噪音（弱关联） | 含义对但关联不深 | 偶发引用、类比使用 |

### 第四步：概念群噪音画像

为每个概念群建立噪音画像：

```
概念群 G[N]: [name]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
检索词：[terms]
检索方式：[method]
命中篇数：[count] ([percent]%)
风险等级：[level]

预测噪音类型：
  第一层：[词形噪音列表]
  第二层：[主题噪音列表]
  第三层：[边界噪音列表]

建议清洗策略：
  [针对性建议]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 第五步：跨概念群交叉分析

- 哪些文献被多个概念群同时命中？（通常更可能相关）
- 哪些文献仅被一个概念群命中？（需要更仔细判定）
- 概念群之间的命中重叠度如何？

## 字段判定权重（通用模板）

根据来源平台的字段结构，建议字段判定优先级：

```
标题 > 作者关键词 > 摘要 > 学科分类 > 期刊名

快速判定路径：
  标题明确噪音 → NOISE
  标题明确相关 → RELEVANT（读摘要确认子类）
  标题不确定 → 读关键词 + 摘要前两句 → 综合判定
  仍不确定 → 读完整摘要 + 学科分类 → PENDING
```

> 权重是建议而非硬规则，具体文献可能需要灵活调整。

```

一、检索式结构
  概念群数量：[N]

二、风险评估
  [各概念群风险等级]
  整体噪音预期：[比例范围估计]

三、噪音类型预测
  [预测的噪音类型列表，含预期数量级]

四、清洗策略建议
  [优先处理哪些噪音类型]
  [建议的分批策略]

→ 建议下一步：S07-分批语义判定
  （如数据集超过上下文窗口容量，先执行 S06-超长文本读取）
```
