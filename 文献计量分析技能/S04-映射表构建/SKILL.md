# S04 映射表构建

## 定位

清洗流水线**操作中枢构建技能**。映射表是整个清洗项目的核心数据结构——所有判定、修订、复核、导出都通过映射表实现。本技能负责首次创建映射表。

## 前置依赖

- S01-来源类型检测 已完成（提供字段解析规则）
- S02-来源数据分析 已完成（提供记录数和字段完整性）
- S03-研究背景问询 已完成（提供判定等级定义）

## 核心原则

1. **seq 不可变**：一旦分配，文献编号不得修改，即使文献被剔除
2. **一表到底**：整个清洗项目只有一张映射表，所有轮次共用
3. **全量覆盖**：每条原始记录必须有对应行，不允许遗漏
4. **修订追溯**：每次修改都在映射表中留下痕迹

## 映射表字段规范

### 通用字段（所有来源平台）

| 字段名 | 类型 | 说明 | 是否可修改 |
|--------|------|------|-----------|
| `seq` | 整数 | 按出现顺序分配的唯一回溯编号 | **不可修改** |
| `source_id` | 字符串 | 来源平台的唯一标识（UT/DOI/PMID/EID等） | 不可修改 |
| `TI` | 字符串 | 标题（便于快速定位） | 不可修改 |
| `PY` | 整数 | 出版年 | 不可修改 |
| `DT` | 字符串 | 文献类型 | 不可修改 |
| `concept_group` | 字符串 | 检索词概念群归组（G1/G2/G3...） | 首轮后可修订 |
| `hit_words` | 字符串 | 实际命中的检索词 | 首轮后可修订 |
| `verdict` | 字符串 | 判定结果 | 每轮可修订 |
| `noise_type` | 字符串 | 噪音类型编码（仅NOISE有值） | 每轮可修订 |
| `reason` | 字符串 | 判定理由 | 每轮追加 |
| `round` | 整数 | 最后判定/修订的轮次 | 自动更新 |

### 扩展字段（多轮清洗追踪）

| 字段名 | 说明 |
|--------|------|
| `noise_type_r[N]` | 第N轮追加的噪音类型（保留历史） |
| `verdict_history` | 判定历史（如 "R1:NOISE→R3:RELEVANT_CORE"） |

### 判定等级（通用模板，根据 S03 研究背景定制）

| 判定 | 语义定义 | 处理 |
|------|----------|------|
| `NOISE` | 与研究主题无实质语义关联 | 剔除 |
| `RELEVANT_CORE` | 目标概念是研究对象或理论框架 | 保留 |
| `RELEVANT_[分支名]` | 与研究主题的某一分支相关（名称根据实际定制） | 保留 |
| `PENDING_LIKELY` | 可能相关，需进一步确认 | 待定 |
| `PENDING_WEAK` | 弱关联，需人工判定 | 待定 |

> 判定等级不是封闭的——发现新类型时应提出新等级并说明理由。

## 构建流程

### 第一步：检索词命中分析

在构建映射表前，分析检索策略中各检索词/概念群的命中情况：

1. 将检索式拆解为独立的概念群（如 G1=meme*，G2=cultural gene，G3=coevolution...）
2. 对每条记录，判定它被哪个/哪些概念群命中
3. 记录实际命中的具体词形

### 第二步：逐条建行

遍历原始数据文件，为每条记录创建映射表行：

- `seq`：从 1 开始递增
- `source_id`：提取平台唯一标识
- `TI`：提取标题
- `PY`/`DT`：提取年份和类型
- `concept_group`/`hit_words`：填入第一步的结果
- `verdict`/`noise_type`/`reason`/`round`：留空（待后续技能填充）

### 第三步：一致性校验

| 校验项 | 方法 |
|--------|------|
| 行数校验 | 映射表行数 = 原始记录数 |
| seq 连续性 | 1 到 N 无跳号无重复 |
| source_id 唯一性 | 无重复标识 |
| 编码校验 | 确认 UTF-8 with BOM |

### 第四步：存档

- 映射表存入 `02_文献分类映射/` 目录
- 文件名格式：`文献映射表_逐篇分类与判定.csv`
- 在清洗日志中记录映射表创建信息

## 后续轮次

后续清洗轮次**不重建映射表**，而是：
1. 加载已有映射表
2. 在对应行更新 `verdict`/`noise_type`/`reason`/`round` 字段
3. 修订前轮判定时，在 `reason` 中注明历史（"第X轮修订，原判定XXX"）

## 输出规范

```
=== 映射表构建报告 ===

映射表路径：[path]
总行数：[count]（含表头）
字段数：[field_count]

概念群命中分布：
  G1 [name]: xxxx 篇 (xx.x%)
  G2 [name]: xxx 篇 (x.x%)
  ...
  未匹配：x 篇 (x.x%)

校验结果：
  ✓ 行数一致
  ✓ seq 连续
  ✓ source_id 唯一
  ✓ 编码正确

→ 建议下一步：S05-检索策略解析
```

## 与后续技能的接口

本技能输出传递给：
- **S05-检索策略解析**：提供概念群命中分布
- **S07-分批语义判定**：提供待判定记录列表
- **S08-人工复核学习**：提供修订入口
- **S10-结果导出**：提供最终判定结果
