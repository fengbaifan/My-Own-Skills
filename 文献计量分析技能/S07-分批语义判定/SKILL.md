# S07 分批语义判定

## 定位

清洗流水线**核心执行技能**。这是实际进行文献噪音判定的技能——逐篇（或逐批）阅读文献内容，结合研究背景和噪音预期，做出保留/剔除的判定。

## 前置依赖

- S03-研究背景问询 已完成（提供判定锚点）
- S04-映射表构建 已完成（提供操作入口）
- S05-检索策略解析 已完成（提供噪音预期）
- S06-超长文本读取 就绪（大文件分块策略）

## 核心原则

### 主题相关性优先

一篇文献是否保留，取决于它是否与用户定义的研究主题存在**实质性语义关联**。

判断优先级：
1. **语义实质** > 关键词表面匹配
2. **研究意图** > 学科归属
3. **边界宽容** > 过度排除（具体宽容程度由 S03 研究目的决定）

### 自适应迭代

清洗是**持续优化过程**，不是一次性动作：
- 每轮清洗后评估质量，识别误判模式
- 新噪音模式归纳为规则，旧规则若失效则废弃
- 后续轮次基于上一轮**保留集**工作

## 分析方法

### 噪音识别三层模型

| 层级 | 类型 | 核心问题 | 判断路径 |
|------|------|----------|----------|
| **第一层** | 词形噪音（通配符溢出） | 文献中是否存在真正的目标概念？ | 无 → 直接 NOISE |
| **第二层** | 主题噪音（同词异义） | 该词在文献语境中指什么？ | 指向非目标领域 → NOISE |
| **第三层** | 边界噪音（弱关联） | 目标概念是研究对象还是偶发引用？ | 偶发引用 → PENDING |

### 相关性多维判定框架

| 维度 | 问题 | 依据 |
|------|------|------|
| **词义** | 命中词的实际含义？ | 标题 + 摘要上下文 |
| **主题** | 核心研究主题？ | 标题 + 关键词 + 摘要首句 |
| **角色** | 目标概念的角色？ | 研究对象 / 理论框架 / 偶然引用 |
| **学科** | 所属学科领域？ | 学科分类 + 期刊名 |
| **贡献** | 对目标领域的知识贡献？ | 综合判断 |

### 快速判定路径

```
标题明确噪音（完全无关词形/学科） → NOISE
标题明确相关（含核心概念且为研究对象） → RELEVANT（读摘要确认子类）
标题不确定 → 读关键词 + 摘要前两句 → 综合判定
仍不确定 → 读完整摘要 + 学科分类 → PENDING
```

## 单轮清洗流程

### 步骤一：确定本轮范围

| 轮次 | 输入 | 处理范围 |
|------|------|----------|
| 首轮 | 原始数据全量 | 所有记录 |
| 后续轮次 | 上一轮保留集 | 上一轮 verdict ≠ NOISE 的记录 |
| 深度清洗 | 指定子集 | PENDING类 / 特定concept_group |
| 概念群专洗 | 指定concept_group | 仅处理被特定概念群命中的记录 |

### 步骤二：制定本轮策略

基于 S05 噪音预期和前轮经验，明确：
1. 本轮优先处理的噪音类型
2. 判定边界和争议处理方向
3. 是否需要新的噪音模式识别
4. 是否扩展字段扫描范围（如加入 DE/ID 字段）

### 步骤三：分批处理

使用 S06-超长文本读取 的分块策略，逐批执行：

**每批流程：**

1. **读取本批文献**（20-50篇的关键字段）
2. **逐篇判定**：
   - 按快速判定路径初筛
   - 对需要深入分析的文献执行多维判定
   - 记录 verdict + noise_type + reason
3. **批次小结**：
   ```
   [批次 M/N] 本批 XX 篇
   NOISE: XX | RELEVANT_CORE: XX | RELEVANT_[分支]: XX | PENDING: XX
   新发现模式：[如有]
   ```
4. **规则迭代**：遇到新模式时立即评估是否需要更新策略

### 步骤四：回写映射表

将判定结果写入映射表对应行：
- 更新 `verdict`、`noise_type`、`reason`、`round` 字段
- 推翻前轮判定时，在 `reason` 中保留历史

### 步骤五：质量检查

本轮全部批次完成后：

| 检查项 | 方法 |
|--------|------|
| 统计各类别篇数和占比 | 是否符合预期？ |
| 抽检 NOISE 文献 | 随机抽 20 篇，有无误判？ |
| 抽检 RELEVANT 文献 | 随机抽 20 篇，有无漏判？ |
| 检查 PENDING 文献 | 数量是否合理？是否需要进一步处理？ |
| 发现系统性偏差？ | 回到步骤二重新制定策略 |

### 步骤六：导出结果

调用 S10-结果导出 生成本轮输出文件。

## 规则自优化机制

```
模式发现 → 规则提炼 → 规则验证 → 入库/退役
```

- 每批处理中留意重复出现的噪音特征
- 反复出现的判定逻辑抽象为命名规则（编码 N1, N2, N3...）
- 新规则经验证后写入 S09-噪音模式库
- 失效规则标记废弃并说明原因

### 概念群感知策略

> concept_group 字段（G1/G2/G3...）标记文献被哪个检索概念群命中，是噪音风险的最强先验信号。后续清洗应优先利用此信息。

**高风险概念群**（如 G1 meme\* 通配符）每轮都应检查：
- 是否有新型词形噪音未被识别？
- 已有噪音规则的边界是否漂移？

**低风险概念群**（如 G2/G3 精确短语）可减少检查频率。

**概念群→噪音类型映射**（由 S05 噪音预期填充，S07 本轮更新）：

```
G1 (meme*)     → 高噪音率，优先多轮深度清洗
G2 (cultural gene等) → 中噪音率，关注词序拼接假碰撞
G3 (gene-culture coevolution) → 低噪音率，关注跨领域同形
G4 (dual inheritance theory) → 极低噪音率，关注综述类假阳性
```

### PENDING 类别处理路径

S04 定义了以下 PENDING 等级：

| 等级 | 语义 | 处理路径 |
|------|------|---------|
| `PENDING_LIKELY` | 较可能相关，待确认 | 下一轮优先处理，尽量给出明确判定 |
| `PENDING_WEAK` | 关联较弱，待确认 | 可批量扫描，或直接给出 NOISE/RELEVANT |
| `PENDING`（无后缀） | 灰色地带，特征不明确 | **必须逐篇人工复核**，不得跳过 |

> 注：`PENDING`（无后缀）是最高风险等级，意味着文献同时触发多个矛盾信号，无法自动判定。处理路径必须包含用户复核步骤（S08）。

## 多轮迭代原则

- 每轮基于上一轮**保留集**工作
- 新轮次目录：按顺序编号（如 `05_噪音去除第五次/`）
- 可推翻前轮判定，但必须说明理由并记录
- 每轮结束更新清洗日志：策略、规则增删、统计、经验教训
- 噪音密度低于阈值（如<1%）时建议结束清洗

### 清洗结束条件

满足以下任一条件时，建议结束清洗：

| 条件 | 说明 |
|------|------|
| 噪音密度阈值 | 本轮新剔除数 < 输入总数的 1% |
| 稳定收敛 | 连续 2 轮 PENDING 类文献数量无显著变化 |
| 质量达标 | S11 批判性审查达到 A 级 |
| 用户确认 | 用户主动确认当前结果为最终版本 |

## 与其他技能的接口

- **输入依赖**：S03（判定锚点）、S04（映射表）、S05（噪音预期）、S06（分块读取）
- **输出传递**：S08-人工复核学习（可疑判定）、S09-噪音模式库（新模式）、S10-结果导出（判定结果）
