# S13 关键词突现分析

## 定位

清洗流水线**后处理分析技能**。本技能不属于清洗流水线，不参与噪音判定或数据清洗。它的作用是在清洗完成后，对保留文献进行文献计量学意义上的**趋势分析**——识别研究主题的演变阶段和新兴热点。

> **与其他技能的关系**：S13 可在任何阶段独立运行。运行结果不写入映射表，不影响清洗判定。S11 批判性审查时，突现分析结果可作为辅助参考。

## 核心概念

### 什么是突现（burst）？

一个关键词在某段时间内的出现频率**显著高于**其长期均值，称为突现。突现意味着该主题在这段时间内受到了特别关注——可能是技术突破、学术争议、政策事件或社会现象驱动。

### Kleinberg 突现检测算法

本技能使用基于 Kleinberg 两状态自动机模型的简化版检测算法：

```
状态0（正常）→ 状态1（突现）→ 状态0（正常）
```

- **基准频率**：关键词在整个时间跨度的平均出现频率
- **突现区间**：连续N年频率显著高于基准的时间段
- **突现强度**：区间内实际频率 / 基准频率的比值

> 算法原理见 [references/kleinberg.md](references/kleinberg.md)。参考实现脚本见 [scripts/burst_detection.py](scripts/burst_detection.py)。

### 第三维：阶段性趋势分析

按突现结束年份分期，划分研究发展阶段。

**默认分期**：

| 阶段 | 定义 |
|------|------|
| 早期探索 | ≤ 2005 |
| 发展期 | 2006 – 2015 |
| 爆发期 | 2016 – 2020 |
| 前沿期 | 2021 – 至今 |

### 第四维：新兴关键词识别

筛选**最近5年内开始突现**的关键词，反映当前研究前沿。

**目的**：识别尚未充分研究的新兴方向，可能反向提示清洗中遗漏的相关文献。

## 分析流程

### 步骤一：提取数据

从输入文件中提取 PY（年份）、DE、ID 字段，构建关键词-年份矩阵。

### 步骤二：构建年度词频矩阵

- 合并 DE 和 ID 字段，统一小写
- 同一篇文献中重复出现的关键词只计一次
- 构建 `{关键词 → {年份 → 频次}}` 的嵌套字典

### 步骤三：计算年度基准

```
基准频率 = Σ(各年频次) / 时间跨度年数
```

### 步骤四：检测突现区间

对每个候选关键词（频次 ≥ 阈值，如 ≥5 次），检测其突现区间：

1. 计算每个滑动窗口的相对频率
2. 标记频率显著高于基准的连续年份
3. 合并相邻突现区间（间隔 ≤1 年视为连续）
4. 计算每个区间的突现强度
5. 去重叠：保留强度最大的非重叠区间

### 步骤五：识别新兴关键词

筛选 `起始年 ≥ 当前年 - 5` 的突现关键词。

### 步骤六：导出结果

输出到 `关键词突现分析结果.csv`：

| 字段 | 说明 |
|------|------|
| 排名 | 按突现强度排序 |
| 关键词 | 小写关键词 |
| 总频次 | 全时间跨度出现总次数 |
| 突现强度 | 区间均值 / 基准均值 |
| 突现起始 | 突现区间开始年 |
| 突现结束 | 突现区间结束年 |
| 持续年数 | 突现窗口长度 |
| 突现期频次 | 突现窗口内出现次数 |

```
[最终数据集目录]/
  ├── 去噪完成_最终保留YYYY篇.txt        ← 输入

```

## 分析模式

| 模式 | 说明 |
|------|------|
| 全面分析 | 高频 + 突现 + 阶段趋势 + 新兴关键词 + 时间线 |
| 突现聚焦 | 仅检测突现关键词（适合快速扫描） |
| 高频排名 | 仅输出 Top N 高频词（适合补全噪音模式线索） |
| 新兴前沿 | 仅识别近5年新兴关键词 |

## 与其他技能的接口

- **前置**：清洗完成（S10 导出后即可运行）
- **辅助 S11**：突现分析结果可作为 S11 批判性审查的学科覆盖参考
- **辅助 S07**：新兴关键词可提示是否遗漏了相关子主题

## 关于参考脚本

本技能的参考实现脚本位于 `scripts/burst_detection.py`（基于 Kleinberg 算法）。该脚本：
- 适用于 WoS 格式数据
- 支持 DE 和 ID 字段提取
- 输出 CSV 和 ASCII 可视化
- 可独立运行：`python burst_detection.py`

运行参考脚本前，需修改 `main()` 函数中的 `base_dir` 和 `input_file` 路径为当前工作区路径。
