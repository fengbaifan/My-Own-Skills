# S03 研究背景问询

## 定位

清洗流水线**语义锚点技能**。文献清洗的本质是"与研究主题不相关的文献识别与剔除"，因此**研究主题的精确定义**是所有判定的基础。本技能通过结构化问答，与用户共同建立清晰、可操作的研究边界。

## 前置依赖

- S01-来源类型检测 已完成
- S02-来源数据分析 已完成（提供数据概况）

## 核心原则

1. **问询而非猜测**：研究主题由用户定义，不由数据内容推断
2. **逐步精确化**：从宽泛到具体，引导用户明确边界
3. **可操作性**：最终输出的定义必须能指导一致性判定
4. **可修订**：用户可以在任何阶段修订研究范围

## 问询流程

### 必答问题组（第一轮）

以下问题**必须**在清洗开始前获得用户回答：

#### Q1：学科领域

> 您的研究属于什么学科领域？
>
> 示例：社会学、传播学、文化研究、进化生物学、计算机科学、交叉学科...

**目的**：建立学科预期，辅助理解边缘文献的归属。

#### Q2：研究主题

> 请用 1-3 句话描述您的核心研究主题。
>
> 示例：
> - "我研究互联网模因(internet meme)的传播机制与文化影响"
> - "我关注基因-文化协同进化理论的发展脉络"
> - "我研究社交媒体上政治幽默的传播与效果"

**目的**：这是所有判定的**最终参照锚点**。

#### Q3：研究目的

> 这批文献数据将用于什么目的？
>
> 选项：
> - [ ] 系统文献综述 (Systematic Review)
> - [ ] 文献计量分析 (Bibliometric Analysis)
> - [ ] 元分析 (Meta-analysis)
> - [ ] 范围综述 (Scoping Review)
> - [ ] 定性分析 / 主题分析
> - [ ] 其他：___

**目的**：不同目的对清洗精度的要求不同。

| 目的类型 | 清洗倾向 | 说明 |
|----------|----------|------|
| 系统综述 | 宁多勿少 | PENDING 倾向保留，避免遗漏 |
| 文献计量 | 精确优先 | 噪音对统计结果影响大，倾向严格剔除 |
| 元分析 | 极高精度 | 须精确匹配纳入标准 |
| 范围综述 | 适度宽容 | 边界文献可保留 |

#### Q4：检索策略

> 请提供您使用的检索式（搜索表达式）。
> 如果有多个平台的检索式，请逐一提供。

**目的**：理解噪音来源——哪些检索词可能引入跨领域文献。

#### Q5：主题边界

> 以下类型的文献，您希望**保留**还是**剔除**？
>
> （根据 S02 数据分析结果动态生成选项。以下为通用示例：）
>
> - 研究主题仅在摘要中被提及一次（作为背景或类比），核心研究不相关
> - 研究主题出现在关键词但摘要实际讨论其他领域
> - 与研究主题相关但属于不同学科分支
> - 书评/综述中讨论了相关主题但本身不构成原创研究
> - 检索词的同形异义词命中（如 meme* 命中了 memento/membrane 等）

**目的**：明确灰色地带的处理方向。

### 条件性问题组（第二轮）

根据第一轮回答和数据分析结果，可能追加：

#### Q6：子主题范围（如研究主题包含多个分支）

> 您的研究是否覆盖以下所有子主题？请勾选需要包含的：
>
> （根据数据中实际发现的子主题动态生成）

#### Q7：时间范围限定

> 是否限定文献的时间范围？
> - [ ] 不限
> - [ ] 仅限 ____年 至 ____年

#### Q8：语言范围

> 是否只保留特定语言的文献？
> - [ ] 仅英文
> - [ ] 英文 + 中文
> - [ ] 全部语言
> - [ ] 其他：___

#### Q9：文献类型范围

> 以下文献类型是否纳入清洗范围？
>
> （基于 S02 检测到的文献类型动态生成）

## 输出规范

问询完成后，生成 **研究背景档案**：

```
=== 研究背景档案 ===

学科领域：[field]
研究主题：[topic_description]
研究目的：[purpose_type]
清洗倾向：[宁多勿少 / 精确优先 / 极高精度 / 适度宽容]

检索策略：
  [search_expression]
  噪音风险点：[根据检索词分析的风险]

主题边界定义：
  核心概念：[core_concepts]
  相关分支：[related_branches]
  明确排除：[excluded_topics]

数据范围：
  时间窗口：[year_range]
  语言范围：[language_range]
  文献类型：[doc_type_range]

灰色地带处理方向：
  - 弱关联文献：[保留为PENDING / 直接剔除]
  - 同形异义：[逐篇判定 / 直接剔除]
  - 非核心分支：[保留 / 需用户确认]
```

## 修订机制

- 用户可在清洗任何阶段要求"重新确认研究背景"
- 修订后，受影响的已判定文献应标记为"待重新评估"
- 修订内容记入清洗日志

## 与后续技能的接口

本技能输出传递给：
- **S04-映射表构建**：提供判定等级定义和字段配置
- **S05-检索策略解析**：提供检索式和噪音风险点
- **S07-分批语义判定**：提供判定锚点和边界定义
- **S08-人工复核学习**：提供判定标准的参照基线
