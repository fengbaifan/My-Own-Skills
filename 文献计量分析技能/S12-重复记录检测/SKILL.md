# S12 重复记录检测

## 定位

清洗流水线**可选辅助技能**。重复记录会影响文献计量分析的准确性。去重不是清洗流水线必需步骤，但当 S02 发现重复时，应执行本技能。

## 前置依赖

- S02-来源数据分析 已完成（提供数据概况）
- S04-映射表构建 已完成（提供 seq 和 source_id 字段）

## 重复检测策略

### 第一层：精确重复

| 判定条件 | 方法 |
|----------|------|
| DOI 精确相同 | DOI 非空且相同 |
| WoS UT 精确相同 | UT 字段相同 |
| PMID 精确相同 | PMID 字段相同 |

**保留策略**：保留记录更完整的那条（字段数更多，或年份更晚）。

### 第二层：疑似重复

| 判定条件 | 方法 |
|----------|------|
| 标题相似度 > 90% | 去掉标点、冠词后比较 |
| 出版年相同 | PY 相同 |

**保留策略**：同精确重复。

### 第三层：版本重复

| 判定条件 | 方法 |
|----------|------|
| 标题相同 | 完全一致（忽略大小写） |
| 期刊名相同 | SO 字段相同 |
| 一种为 Early Access | DT 含 "Early Access" 或 "Article" 而发表年不同 |

**保留策略**：保留正式出版版本（DT 为 "Article" 且有正式发表年）。

## 检测流程

### 步骤一：精确重复扫描

对所有记录按 source_id（如 DOI/UT）分组：

```
同一 source_id 出现 ≥2 条 → 标记为精确重复组
```

输出重复组列表。

### 步骤二：疑似重复扫描

对无 source_id 或 source_id 相同的记录，按"标题+年份"比对：

```
相同标题 + 相同年份 → 标记为疑似重复组
```

### 步骤三：版本重复扫描

对疑似重复组，进一步检查是否有版本关系（正式版 vs Early Access）。

### 步骤四：呈现与确认

向用户展示重复组列表：

```
发现 X 组重复（Y 篇文献）：

组1 (2篇)：
  - seq=001, "Title A", PY=2020, DOI=10.xxx/xxx  [→ 保留]
  - seq=042, "Title A", PY=2020, DOI=null      [→ 剔除]

组2 (3篇)：
  ...
```

### 步骤五：用户决策

| 选项 | 说明 |
|------|------|
| 自动去重 | 按保留策略自动处理，映射表标记被剔除者 |
| 人工确认 | 每组逐一确认保留哪条 |
| 仅报告 | 不执行剔除，仅报告重复模式 |

### 步骤六：更新映射表

去重决定写入映射表：

| 字段 | 值 |
|------|------|
| `verdict` | 设为 `DUPLICATE` |
| `reason` | 注明原始 seq，如 "重复 seq=001" |
| `round` | 当前轮次 |

> `DUPLICATE` 视为一种特殊 verdict，在 S10 导出时与 NOISE 同样处理（不保留在输出中）。

## 与其他技能的接口

- **前置**：S02（发现重复时触发）、S04（获取 source_id）
- **输出**：映射表更新，S10 导出时自动过滤 DUPLICATE 记录
- **触发**：用户主动要求去重，或 S02 数据分析发现重复模式
