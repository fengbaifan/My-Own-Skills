# S02 来源数据分析

## 定位

清洗流水线**数据画像技能**。在来源类型确定后，对数据集进行全面的结构化扫描，建立数据画像。本技能回答的核心问题是：**这批数据长什么样？质量如何？有多少可以用？**

## 前置依赖

- S01-来源类型检测 已完成
- 已确定：来源平台、导出格式、字段映射

## 分析流程

### 第一步：全量记录扫描

> **注意**：大文件须调用 S06-超长文本读取 的分块策略，确保全量覆盖。

逐条扫描所有记录，采集以下元数据：

| 采集项 | 说明 |
|--------|------|
| 总记录数 | 精确计数，与文件标称数核对 |
| 记录编号分配 | 按出现顺序分配 seq（1, 2, 3...），不可变 |
| 各字段存在率 | 每个字段在多少条记录中有值 |

### 第二步：字段完整性检查

按来源平台的标准字段集，逐字段统计：

```
字段完整性报告
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
字段       有值记录数    缺失记录数    完整率
─────────────────────────────────────────
[核心字段]
标题       xxxx          x            99.9%
摘要       xxxx          xxx          95.2%
作者       xxxx          xx           98.5%
关键词     xxxx          xxxx         72.3%
...
[辅助字段]
学科分类   xxxx          xxx          93.1%
期刊名     xxxx          x            99.8%
DOI        xxxx          xxxx         78.6%
...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**字段分级**（通用标准，根据来源平台调整）：

| 级别 | 含义 | 清洗中的角色 |
|------|------|-------------|
| **核心字段** | 标题、摘要 | 语义判定的主要依据，缺失需标记 |
| **关键字段** | 作者关键词、分类号 | 辅助判定和分析的重要字段 |
| **辅助字段** | 期刊名、学科分类、DOI、年份 | 提供上下文和回溯能力 |
| **索引字段** | 唯一标识（UT/PMID/EID等） | 去重和回溯的基础 |
| **冗余字段** | 出版商地址、资助信息等 | 清洗中通常不用 |

### 第三步：文献类型分析

统计文献类型（Document Type / DT）的分布：

| 类型 | 说明 | 清洗提示 |
|------|------|----------|
| **Article** | 原创研究论文 | 清洗核心对象 |
| **Review** | 综述论文 | 通常保留，但需注意引用meme/memetics的综述未必以此为主题 |
| **Book Review** | 书评 | 可能提及相关概念但本身不构成研究贡献，需单独判定策略 |
| **Editorial Material** | 编辑评论 | 短文，信息量有限，需评估 |
| **Letter** | 读者来信 | 同上 |
| **Conference Paper** | 会议论文 | 与 Article 同等对待 |
| **Book Chapter** | 书章 | 需评估完整性 |
| **News Item** | 新闻报道 | 通常非学术研究，需评估 |
| **Correction / Retraction** | 勘误/撤稿 | 特殊处理——撤稿标记需传递到原文 |
| **Early Access** | 预印本/早期获取 | 正常对待，但注意可能与正式版重复 |

**输出格式：**
```
文献类型分布
━━━━━━━━━━━━━━━━━━━━━━
类型                 数量    占比
────────────────────────────
Article              xxxx    xx.x%
Review               xxx     x.x%
Book Review          xx      x.x%
...
━━━━━━━━━━━━━━━━━━━━━━
```

### 第四步：语言分布分析

统计文献语言（Language / LA）分布：

- 全英文数据集：确认后简要报告
- 多语言数据集：详细列出各语言篇数和占比，标注需要特殊处理的语言
- **关键提示**：非英语文献可能导致同形异义噪音（如法语 même、土耳其语 meme = 乳腺、印尼语 meme* 前缀词）

### 第五步：时间分布分析

统计出版年（Publication Year / PY）分布：

```
年度分布
━━━━━━━━━━━━━━━━━━━━━━
年份    篇数    ████████（柱状图）
────────────────────────
[min]   x
...     ...
[max]   xxx     ████████████████
━━━━━━━━━━━━━━━━━━━━━━
```

标注：
- 最早年份 / 最晚年份 / 峰值年份
- 是否存在异常年份（如远早于领域起源的文献、未来年份的文献）

### 第六步：重复记录检测

检测可能的重复记录：

| 检测策略 | 方法 |
|----------|------|
| 精确重复 | 唯一标识（UT/DOI/PMID）完全相同 |
| 疑似重复 | 标题相似度 > 90% 且年份相同 |
| 版本重复 | Early Access + 正式版并存 |
| 跨库重复 | 多来源合并时，同一文献出现在不同库中 |

### 第七步：异常记录标记

扫描并标记以下异常：

| 异常类型 | 识别方法 |
|----------|----------|
| 空记录 | 仅有标识，无实质字段 |
| 截断记录 | 字段内容在中间断开（编码错误或导出截断） |
| 格式错误 | 字段标签缺失或错位 |
| 撤稿标记 | 标题含 "Retracted" / "RETRACTED" / "WITHDRAWN" |
| 勘误记录 | DT = Correction 或标题含 "Erratum" / "Corrigendum" |

## 用户问答环节

在数据分析过程中或完成后，可能需要向用户确认：

1. **文献类型处理策略**：书评/编辑评论/来信是否纳入清洗范围？还是直接排除？
2. **语言范围**：是否只保留英文文献？其他语言如何处理？
3. **时间范围**：是否限定时间窗口？过早/过晚的文献如何处理？
4. **重复处理**：发现 X 组重复记录，是否自动去重？保留哪个版本？
5. **撤稿处理**：发现 X 篇撤稿文献，是否直接排除？

## 与后续技能的接口

本技能输出传递给：
- **S03-研究背景问询**：提供数据概况，辅助用户确认清洗范围
- **S04-映射表构建**：提供字段列表和记录编号
- **S06-超长文本读取**：反馈实际记录数和文件结构，优化分块参数
