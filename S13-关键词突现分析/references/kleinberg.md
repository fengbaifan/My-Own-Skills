# Kleinberg 突现检测算法

## 原始算法（Kleinberg, 2002）

Kleinberg 在《Bursty and Hierarchical Structure in Streams》中提出了一种基于有限状态自动机的突现检测方法。

### 基本思想

为每个关键词构建一个两状态自动机：
- **状态0（正常）**：关键词以基准概率出现
- **状态1（突现）**：关键词以高概率出现

自动机的最优状态序列通过 Viterbi 算法求解，使得整体代价最小。

### 核心参数

| 参数 | 说明 | 典型值 |
|------|------|--------|
| `s` | 状态转换比率（正常→突现的概率放大倍数） | 2.0–3.0 |
| `gamma` | 状态转换惩罚（防止频繁切换） | 0.5–1.0 |

### 代价函数

从状态0转换到状态1时：
```
cost = gamma * log(s)
```

在状态1中产生观测值 x 时：
```
cost = -log(P(x | state=1))
```

在状态0中产生观测值 x 时：
```
cost = -log(P(x | state=0))
```

## 本技能使用的简化版本

### 简化策略

原始 Viterbi 算法对长序列和大量关键词的计算开销较大。本技能使用基于滑动窗口的简化方法：

1. **计算基准频率**：整个时间跨度的平均频率
2. **滑动窗口检测**：对每个可能的时间窗口 [start, end]，计算窗口内均值
3. **突现判定**：窗口均值 > 基准 × 阈值（默认 2.0）且窗口内频次 ≥ 3
4. **去重叠**：多窗口重叠时，保留突现强度最大的

### 突现强度定义

```
突现强度 = 窗口内相对频率均值 / 全局相对频率均值
        = (窗口内频次 / 窗口年数) / (总频次 / 总年数)
```

### 简化版本的局限性

- 不做状态序列优化（无 Viterbi）
- 无法区分"真正的突现"和"长期上升趋势"
- 无法检测孤立单年尖峰（要求持续 ≥ 2 年）

### 何时使用完整版

如果需要更严格的突现检测（如论文发表），建议：
1. 手动审查突现结果，排除长期趋势项
2. 使用原版 Kleinberg 实现（如 Python 的 `bursts` 库）
3. 结合 CiteSpace 的突现检测功能交叉验证
