# S08 人工复核学习

## 定位

清洗流水线**学习引擎**。它不执行清洗本身，而是从用户的每一个人工决定中提取知识，推动筛选策略的持续进化。

## 核心理念

```
用户的人工决定 ≠ 孤立的个案修正
用户的人工决定 = 对筛选策略的隐式反馈信号
```

每当用户确认、推翻、或修改一个判定，背后都隐含着一条筛选逻辑。本技能的任务是**把隐式反馈变成显式的策略优化**。

## 工作流程

### 阶段 A：复核执行

1. **确定复核范围**
   - 用户指定：特定噪音类型 / PENDING 状态 / seq 编号范围 / 随机抽样
   - 或由系统建议：上一轮质量检查中发现的可疑批次

2. **逐篇呈现**
   - 展示关键信息：seq, 标题, 关键词, 摘要（前3句）, 学科分类, 期刊
   - 展示当前判定：verdict, noise_type, reason
   - 给出语义分析意见：当前判定为何可能正确/可能有误

3. **记录用户决定**
   - 用户对每篇的决定：✓ 确认 / ✗ 推翻（改为什么）/ ? 暂缓
   - 用户的理由（哪怕一句话）
   - 立即回写映射表，在 reason 中标注"人工复核，第X轮"

### 阶段 B：推理分析

每完成一批复核（10-20 篇）后：

1. **统计决定分布**
   - 确认/推翻/暂缓各多少篇？
   - 推翻方向：NOISE→RELEVANT 还是 RELEVANT→NOISE？

2. **归因分析**（对每个被推翻的判定）
   - **是规则缺陷？** 规则本身定义有漏洞或过于宽泛/狭窄？
   - **是边界偏移？** 规则没问题，但判定边界需要调整？
   - **是新模式？** 现有规则未覆盖的噪音/相关文献类型？
   - **是个案特例？** 不构成模式，只是单篇的特殊情况？

3. **模式聚类**
   - 将同类推翻案例聚在一起
   - 提取共同特征（学科、关键词模式、期刊类型等）
   - 判断是否构成可复用的规则修订

### 阶段 C：策略优化

1. **提出修订建议**
   ```
   规则修订建议
   ━━━━━━━━━━━━━━━━━━━━━━━━━━
   来源：本轮复核第 X-Y 批
   
   建议1：[修订/新增/废弃] 规则 [编码]
     原因：[基于哪些人工决定推理得出]
     影响范围：预计影响 ~XX 篇
   
   建议2：...
   ━━━━━━━━━━━━━━━━━━━━━━━━━━
   ```

2. **用户确认**：修订建议须经用户确认后才执行
3. **执行修订**：更新 S09-噪音模式库，回溯受影响的已判定文献
4. **记录到清洗日志**

## 与其他技能的接口

- **输入来源**：S07（判定结果）、用户人工决定
- **输出传递**：S09-噪音模式库（规则修订）、S07-分批语义判定（策略更新）
