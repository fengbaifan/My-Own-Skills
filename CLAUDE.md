# bib-clean 文献计量数据清洗框架

## 技能定位

本工作区提供**bib-clean**技能支持，用于对学术数据库（WoS/CNKI/Scopus/PubMed等）导出的文献集进行噪音分析和清洗。

## 技能流水线（13 个编号技能）

```
┌─────────────── 准备阶段 ───────────────┐
│  S01 来源类型检测 → S02 来源数据分析    │
│  S03 研究背景问询 → S04 映射表构建      │
│  S05 检索策略解析                       │
└─────────────────────────────────────────┘
         ↓
┌─────────────── 执行阶段 ───────────────┐
│  S06 超长文本读取（基础设施，按需调用）  │
│  S07 分批语义判定（核心清洗循环）       │
│  S12 重复记录检测（可选，随时可触发）   │
└─────────────────────────────────────────┘
         ↓ ↑（迭代）
┌─────────────── 优化阶段 ───────────────┐
│  S08 人工复核学习 ↔ S09 噪音模式库     │
└─────────────────────────────────────────┘
         ↓
┌─────────────── 收尾阶段 ───────────────┐
│  S10 结果导出与日志 → S11 批判性审查    │
└─────────────────────────────────────────┘
         ↓
┌─────────────── 后处理阶段 ─────────────┐
│  S13 关键词突现分析（独立运行）         │
└─────────────────────────────────────────┘
```

## 目录结构约定

```
[项目工作区]/
├── 01_原始数据/                         ← 严禁修改
├── 02_文献分类映射/                     ← 映射表（操作中枢）
├── 03_噪音去除第一次/                   ← 第一轮
├── 04_噪音去除第二次/                   ← 第二轮
├── 05_噪音去除第三次/                   ← 第三轮
├── 06_噪音去除第四次/                   ← 第四轮
├── (N+2)_去噪完成_最终数据集/          ← 最终输出
├── 00_清洗日志_修改原则与操作记录.md   ← 操作日志
└── .claude/skills/bib-clean/           ← 技能定义
```

## 核心原则

| # | 规范 |
|---|------|
| 1 | 原始数据严禁修改 |
| 2 | 映射表为唯一操作入口 |
| 3 | 结果始终另存新目录 |
| 4 | 纯语义分析，禁止写代码（后处理脚本除外） |
| 5 | 每次变更必须记录到清洗日志 |
| 6 | CSV 编码：UTF-8 with BOM |
| 7 | seq 编号建立后不可变 |
| 8 | 目录编号连续递增，不跳号 |

## 路由指令

当用户提供文献数据或提到以下关键词时，自动路由到对应技能：

| 用户意图 | 路由目标 |
|----------|----------|
| "开始清洗" / "新项目" / 提供数据文件 | S01 → 全流程 |
| "继续清洗" / "下一轮" | S07-分批语义判定 |
| "复核" / "检查判定" | S08-人工复核学习 |
| "审查" / "质量检查" | S11-批判性审查 |
| "导出" / "生成结果" | S10-结果导出与日志 |
| "噪音模式" / "规则" | S09-噪音模式库 |
| "去重" / "重复记录" | S12-重复记录检测 |
| "突现" / "趋势" / "热点" / "研究前沿" | S13-关键词突现分析 |
| "数据概况" / "数据画像" | S02-来源数据分析 |
| 引用 S[编号] | 直接路由到对应技能 |
