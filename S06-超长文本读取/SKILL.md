# S06 超长文本读取

## 定位

清洗流水线**基础设施技能**。大模型上下文窗口有限，但文献清洗要求**零遗漏**。本技能提供可靠的分块读取策略，确保所有记录被完整处理。

## 适用场景

| 场景 | 触发条件 |
|------|----------|
| S02 数据分析 | 全量字段统计 |
| S04 映射表构建 | 全量记录扫描 |
| S07 分批语义判定 | 逐批读取待判定文献 |
| S10 结果导出 | 按判定提取文献 |
| 任何全量操作 | 文件大于上下文窗口容量 |

## 核心原则

1. **零遗漏保证**：每条记录都必须被处理，不允许跳过
2. **边界安全**：分块点必须在记录边界上，不得切割记录
3. **进度可追踪**：每批处理后报告进度
4. **可恢复**：中断后能从断点继续

## 分块策略

### 策略选择矩阵

| 文件格式 | 记录分隔符 | 分块方法 |
|----------|-----------|----------|
| WoS 纯文本 | `ER` 行 | 按 ER 标记分块 |
| CSV | 换行符 | 按行号分块 |
| RIS | 空行 | 按空行分块 |
| BibTeX | `@` 开头行 | 按 @ 标记分块 |
| CNKI 自定义 | 记录标签 | 按记录起始标签分块 |
| XML | 记录元素闭合标签 | 按 XML 元素分块 |

### 分块参数计算

```
安全批量大小 = min(
    上下文窗口容量 × 0.6 ÷ 平均记录长度,
    目标处理效率批量（如50条）
)
```

- **上下文窗口 60% 原则**：读取内容不超过窗口的 60%，留 40% 给分析和输出
- **平均记录长度**：由 S02 的数据分析提供，或在首批读取时估算
- **批量上限**：通常 30-80 条/批，取决于记录的平均长度

### 行号范围读取法

对于大型文件，使用行号范围进行精确分块读取：

```
第1批：第 1 行 → 第 X 行（含完整记录边界）
第2批：第 X+1 行 → 第 Y 行
...
第N批：第 Z+1 行 → 末尾
```

**关键**：每批读取后，检查末尾是否在记录边界上：
- WoS：最后一行是否为 `ER`？如不是，向后扩展到下一个 `ER`
- CSV：最后一行是否为完整行？
- BibTeX：最后一个 `}` 是否闭合？

## 操作流程

### 第一步：文件参数评估

```
文件大小 → 估算总行数 → 计算分块数 → 确定每批行号范围
```

### 第二步：创建读取计划

```
读取计划
━━━━━━━━━━━━━━━━━━━━━━━━━━
文件：[path]
总大小：[size]
估算记录数：[count]
分块策略：[strategy]
批量大小：[batch_size] 条/批
预计批次数：[total_batches]
━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 第三步：逐批执行

每批执行：

1. **读取**：按行号范围读取文件内容
2. **边界校正**：确认读取内容在记录边界上开始和结束
3. **处理**：执行调用方技能要求的操作（如字段提取、语义判定等）
4. **记录进度**：
   ```
   [批次 M/N] 已处理 XXXX/YYYY 条，当前行号 ZZZZ
   ```
5. **衔接**：记录下一批的起始位置

### 第四步：完成校验

全部批次完成后：
- 已处理记录数 = 预期总记录数？
- 是否有跳过的行号范围？
- 是否有重复处理的记录？

## 中断恢复机制

如果处理中断（超时、错误等）：

1. **检查进度**：最后完成的批次编号和行号
2. **确认数据**：已处理的记录是否完整保存
3. **从断点继续**：从下一批开始，不重复处理已完成批次

## 与其他技能的协作方式

本技能**不独立使用**，而是被其他技能调用：

```
调用方技能
  ├─ 准备读取请求（文件路径 + 处理逻辑）
  ├─ S06 计算分块参数
  ├─ S06 逐批读取
  │    └─ 调用方处理每批数据
  └─ S06 完成校验
```

调用方需提供：
- **处理函数**：每批数据要执行什么操作
- **累积规则**：批次间结果如何汇总
- **异常处理**：遇到异常记录如何处理

## 性能提示

- 纯统计操作（计数、字段提取）：可用较大批量（60-80条）
- 语义分析操作（内容判定）：批量缩小（20-40条）
- 复杂推理操作（噪音模式识别）：批量再缩小（10-20条）

### 关于代码辅助的说明

清洗流水线的核心原则是"纯语义分析，禁止写代码"。S06 作为基础设施技能，在以下场景**不限制**使用代码辅助，但必须经过用户确认：

| 可使用代码辅助的场景 | 禁止使用代码辅助的场景 |
|---------------------|----------------------|
| 字段统计、计数、聚合（作为 S02 的辅助） | 语义判定、噪音识别 |
| 分块边界计算（不涉及内容分析） | 主题分类、相关性判断 |
| 批量文本提取（作为 S06 分块的辅助） | 规则生成、模式归纳 |

> 代码辅助脚本（如 `burst_detection.py`）属于**后处理分析工具**，不是清洗流水线的一部分，不受"禁止写代码"原则约束。运行方式见 [S13-关键词突现分析/SKILL.md](S13-关键词突现分析/SKILL.md)。结果不写入映射表。
