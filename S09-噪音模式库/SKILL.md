# S09 噪音模式库

## 定位

清洗流水线**知识沉淀技能**。所有在清洗过程中发现的噪音模式都归档在此，形成可复用的模式库。

## 管理原则

1. 每个模式必须有**真实案例**支撑
2. 模式从**实际数据**中归纳，不是预设的
3. 模式之间可能交叉，**优先语义判断**而非机械匹配
4. 误判率 >10% 时应修订或拆分
5. 废弃模式不删除，标记 `[已废弃]` 并注明原因

## 模式记录模板

每个噪音模式按以下结构记录：

```markdown
## N[编号]: [名称]

- **状态**: 活跃 / 待验证 / 已废弃
- **层级**: 第一层（词形） / 第二层（主题） / 第三层（边界）
- **识别特征**: 什么样的文献会触发此模式？
  - 字段特征（标题/关键词/摘要中的关键词组合）
  - 学科特征（学科分类/期刊领域）
  - 语言特征（特定语言标记）
- **语义判断**: 如何通过阅读内容确认？
- **边界条件**: 哪些看似匹配但不应判为噪音的例外？
- **典型案例**: 2-3 个标题
- **发现轮次**: 第几轮 / 来源数据集
- **累计剔除数**: 
- **修订记录**: 变更历史
```

## 模式分类框架（通用）

以下是常见噪音类型的分类框架，具体模式从数据中归纳：

### 第一层：词形噪音（通配符溢出）

| 类别 | 说明 | 示例 |
|------|------|------|
| 词根扩展 | 通配符命中目标词的无关扩展形式 | meme* → memento, membrane |
| 跨语言同形 | 目标词在其他语言中的非目标含义 | meme = 法语même, 土耳其语"乳腺" |
| 拼写近似 | OCR错误或拼写变体 | measurements → memes |
| 缩写冲突 | 目标词是某个缩写 | MEME = 生信工具 / 培养基 |

### 第二层：主题噪音（同词异义）

| 类别 | 说明 | 示例 |
|------|------|------|
| 跨学科同义词 | 词形相同但学科含义不同 | memetic algorithm ≠ 文化模因 |
| 品牌/专名 | 目标词恰好是品牌名/地名/人名 | Meme District (喀麦隆) |
| 偶然相邻 | 检索短语中的词偶然出现在同一语境 | "cell culture" + "gene delivery" |

### 第三层：边界噪音（弱关联）

| 类别 | 说明 | 示例 |
|------|------|------|
| 偶发引用 | 目标概念仅在摘要中作为类比或背景提及 | "like a meme, this idea spread..." |
| 综述覆盖 | 综述中广泛引用包含目标概念但非核心 | 文化理论综述顺带提及 memetics |
| 模糊边界 | 与研究主题有关但不构成实质贡献 | 用模因做类比但不研究模因本身 |

## 模式库维护操作

### 新增模式

1. 在 S07 分批判定中发现重复出现的噪音特征
2. 抽取 3+ 案例验证一致性
3. 按模板填写模式记录
4. 编码为 N[下一个序号]
5. 状态设为"待验证"
6. 经 1-2 批次验证后改为"活跃"

### 修订模式

1. S08 人工复核发现规则误判
2. 分析误判原因（过宽/过窄/边界不清）
3. 修订识别特征或边界条件
4. 在修订记录中说明变更内容和原因

### 废弃模式

1. 模式不再适用于当前数据集或研究主题
2. 标记状态为"已废弃"
3. 注明废弃原因和替代模式（如有）
4. **不删除**，保留历史参考

## 跨项目复用

模式库中的模式分为：
- **通用模式**：适用于任何文献清洗项目（如 OCR 错误、缩写冲突）
- **领域模式**：适用于特定研究领域（如 meme*/memetic 的同形异义）
- **数据集模式**：仅适用于特定数据集（标记数据集来源）

### 与清洗日志的同步机制

> 本项目已完成 6 轮清洗，发现了 22 个噪音模式（N1-N22）。这些模式的完整记录见 `00_清洗日志_修改原则与操作记录.md` 的"3.1 噪音识别规则"节。

**加载规则**：
- 继续本项目时，从清洗日志加载已有模式到 S09
- 新开项目时，S09 从空状态开始

**模式存档格式**（可导出到清洗日志或导入到新项目）：

```json
{
  "code": "N1",
  "name": "Memetic算法/优化",
  "layer": 1,
  "status": "active",
  "discovered_round": 1,
  "features": ["期刊含优化/AI领域", "DE含algorithm/metaheuristic", "WC为计算机/工程"],
  "examples": ["memetic algorithm", "memetic computing"],
  "cumulative_count": 3139,
  "precision": 99.6
}
```

**归档规则**：
- 每个噪音模式发现后，同步写入清洗日志的模式表
- 清洗日志是模式的持久化存储；S09 是当前会话的工作副本
- 每次 S10 导出时，从 S09 同步最新累计剔除数到清洗日志

## 与其他技能的接口

- **输入来源**：S07（新模式发现）、S08（规则修订）
- **输出传递**：S07（清洗规则参考）、S10（噪音类型编码）
